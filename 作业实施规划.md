# 《智能传感与检测系统》第二个大作业 — 实施规划

## 整体架构

需要创建 **1个ROS2工作空间**，内含 **4个功能包**：

```
ros2_ws/
├── src/
│   ├── robot_description/     ← URDF/XACRO 机器人模型
│   ├── gazebo_scene/          ← Gazebo 世界文件 + launch
│   ├── slam_bringup/          ← Cartographer 配置 + launch
│   └── nav_bringup/           ← Nav2 配置 + launch
├── maps/                      ← 保存的地图 (map.pgm + map.yaml)
├── urdf/                      ← 单独提取的 URDF/XACRO
├── launch/                    ← 单独提取的所有 launch 文件
├── config/                    ← 参数配置文件
└── README.txt
```

## 阶段一：工作空间与机器人建模（预计工作量：25%）

### 步骤 1.1：创建 ROS2 工作空间

- 用 `ros2 pkg create` 创建上述4个功能包（推荐 Python——配置类工作不需要编译复杂逻辑）
- 确定 ROS2 发行版（Humble 或 Jazzy，取决于 Ubuntu 版本）

### 步骤 1.2：编写 URDF/XACRO 机器人模型

> ⚠️ 建议直接用 **XACRO**——参数化写轮子坐标、质量等，后面改参数不用到处改。

需要定义的结构：

| Link | 形状 | 说明 |
|------|------|------|
| `base_link` | 长方体 0.5×0.3×0.25m | 主体，蓝色 |
| `left_wheel_link` | 圆柱体 φ0.15m, 宽0.05m | 左侧驱动轮 |
| `right_wheel_link` | 同上 | 右侧驱动轮 |
| `caster_wheel_link` | 圆柱体 φ0.1m, 宽0.05m | 后方随动轮 |
| `laser_link` | 小圆柱体（示意） | 激光雷达安装位 |

关键 Joint：

| Joint | 类型 | 父→子 | 说明 |
|-------|------|-------|------|
| `base_to_left_wheel` | continuous | base_link → left_wheel_link | 差速左轮 |
| `base_to_right_wheel` | continuous | base_link → right_wheel_link | 差速右轮 |
| `base_to_caster` | fixed | base_link → caster_wheel_link | 随动轮只支撑 |
| `base_to_laser` | fixed | base_link → laser_link | 激光雷达 |

**惯性参数**：每个 link 需要 `<inertial>` 标签（mass + inertia matrix），Gazebo 仿真必需。总质量 2kg，合理分配各组件质量。

**Gazebo 插件**（嵌入URDF中）：
- `libgazebo_ros_diff_drive.so` —— 差速驱动插件（配置轮距、轮径、速度限制 0.5m/s / 1rad/s）
- `libgazebo_ros_ray_sensor.so` —— 激光雷达插件（配置 `/scan` 话题、360°、0.1~5m、10Hz）

### 步骤 1.3：RViz 验证模型

- 写一个 `display_robot.launch.py`：启动 `robot_state_publisher` + `joint_state_publisher_gui` + RViz2
- 在 RViz 中检查：坐标系树（TF）、模型外观、关节可动性
- 用 `ros2 run tf2_tools view_frames` 导出 TF 树截图，后续放入报告

## 阶段二：Gazebo 场景搭建（预计工作量：20%）

### 步骤 2.1：编写 Gazebo 世界文件 (.world)

场景布局（俯视图）：

```
y=5 ┌──────────────────────────────────────┐
    │                                      │
y=3.1│ 入口        障碍物区        出口     │
    │ (0.5,1.9)  ┌─┬─┐  ┌─┐  ╭─╮  (7.5,3.1)│
y=2.5│           │1│2│  │5│  │⊙│           │
    │           └─┴─┘  └─┘  ╰─╯           │
y=1.9│ (0.5,1.9)                         (7.5,1.9)│
    │                                      │
y=0 └──────────────────────────────────────┘
    x=0        x=2.5  x=4.5  x=6.0      x=8
```

**地面**：Box 8×5×0.05m，灰色水泥纹理（Gazebo 内置 `Gazebo/Grey` material）

**5个障碍物**——直接用 SDF `<model>` 标签：
1. Box 0.6×0.6×0.6m @ (2.5, 1.5, 0.3)
2. Box 0.6×0.6×0.6m @ (2.5, 3.5, 0.3)
3. Box 0.8×0.4×0.3m @ (4.5, 2.0, 0.15)
4. Box 0.8×0.4×0.3m @ (4.5, 3.0, 0.15)
5. Cylinder φ0.5m, h=0.3m @ (6.0, 2.5, 0.15)

全部黑色塑料质感（`Gazebo/Black`）

**入口/出口**：用矮 Box 标示通道边界，或在地面上放置浅色矩形标记。

### 步骤 2.2：编写 Gazebo 启动 launch 文件

- `spawn_scene.launch.py`：启动 Gazebo（空世界 or .world 文件）→ 加载场景 models → spawn 机器人
- 机器人出生点：入口处 (x=0.5, y=2.5, z=0.2)，朝向 +x

### 步骤 2.3：验证

- 启动 Gazebo，检查：机器人不穿透地面、激光雷达 topic `/scan` 有数据、障碍物位置正确
- `ros2 topic echo /scan` 验证激光数据
- `ros2 topic echo /odom` 验证里程计

## 阶段三：激光雷达 & SLAM 建图（预计工作量：25%）

### 步骤 3.1：选择 SLAM 方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Cartographer** ⭐推荐 | 精度高、闭环好 | 配置稍复杂，编译耗时 |
| slam_toolbox | 轻量、纯 ROS2 | 需要手动回环 |
| GMapping | 课程提及 | ROS2 支持差，不推荐 |

**第一选择 Cartographer，备选 slam_toolbox。**

### 步骤 3.2：配置 SLAM

- 创建 Cartographer `.lua` 配置文件：
  - `tracking_frame = "base_link"`
  - `published_frame = "odom"`
  - `map_frame = "map"`
  - 激光雷达数据从 `/scan` 读取
  - 机器人尺寸 0.5×0.3m 用于设置子图大小
- 写 `slam.launch.py` 启动 Cartographer 节点

### 步骤 3.3：手动建图

- 同时启动：Gazebo场景 + 机器人 + Cartographer + RViz2
- 用 `teleop_twist_keyboard` 手动控制机器人缓慢移动（线速度 ≤0.3m/s，角速度 ≤0.5rad/s）
- 在场景中走"S形"覆盖所有区域，让 Cartographer 充分扫描障碍物轮廓
- RViz2 中观察实时地图构建

### 步骤 3.4：保存地图

```bash
ros2 run nav2_map_server map_saver_cli -f ~/ros2_ws/maps/map
```

产出 `map.pgm` + `map.yaml`

## 阶段四：路径规划与跟踪（预计工作量：20%）

### 步骤 4.1：配置 Nav2

Nav2 需要的配置文件：

| 文件/节点 | 作用 |
|-----------|------|
| `nav2_params.yaml` | 全局/局部规划器、控制器、行为树、costmap 参数 |
| `planner_server` | 全局路径规划器（NavFn 或 SmacPlanner） |
| `controller_server` | 局部控制器（DWB 或 RPP，差速驱动跟踪） |
| `behavior_server` | 行为树（recovery behaviors：旋转恢复、后退恢复） |
| `bt_navigator` | 行为树 XML |

关键参数：
- 全局 costmap：8×5m，分辨率 0.05m
- 局部 costmap：3×3m 滑动窗口，分辨率 0.05m
- 机器人半径 0.25m（取对角线的一半加余量）
- 膨胀层 inflation_radius 0.3m

### 步骤 4.2：编写导航 launch 文件

- `nav.launch.py`：启动 map_server（加载地图）→ AMCL 定位 → Nav2 各 server → RViz2
- 设置初始位姿：`(x=0.5, y=2.5, yaw=0)`，朝向 +x

### 步骤 4.3：设置目标点

- RViz2 中使用 "Nav2 Goal" 按钮点击出口位置 `(x=7.5, y=2.5)`
- 或写脚本用 `ros2 action send_goal /navigate_to_pose` 发目标

### 步骤 4.4：验证

- 机器人自主规划路径，避开5个障碍物，到达出口
- 路径应呈 S 形绕过障碍物区
- 如果碰撞：调大 inflation_radius 或降低速度

## 建议的加分项（选1-2个）

考虑到难度和收益，推荐：

1. **路径优化**（3-5分）—— 对比 A* vs Dijkstra，在报告中画图说明差异，Nav2 里切换 planner 即可，几乎零代码
2. **可视化优化**（3-5分）—— RViz 中加轨迹 Marker、目标点标注，纯配置
3. **动态避障**（5-8分）—— 在 Gazebo 场景中加一个缓慢移动的障碍物，验证 Nav2 局部规划器的实时避障能力，需要写一个简单的障碍物移动脚本
